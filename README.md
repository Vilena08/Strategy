# Strategy
Стратегия — это поведенческий паттерн проектирования, который определяет семейство схожих алгоритмов и помещает каждый из них в собственный класс, после чего алгоритмы можно взаимозаменять прямо во время исполнения программы.
(из книги "Погружение в паттерны проектирования")

Обладает следующими характеристиками:
Тип: поведенческий.

Принцип работы:

1. Создается интерфейс стратегии, который определяет метод, используемый для выполнения алгоритма.

2. Создаются конкретные классы стратегий, которые реализуют интерфейс стратегии и предоставляют различные реализации алгоритма.

3. Клиентский класс создает объект стратегии и делегирует ему выполнение алгоритма.

4. Клиентский класс может переключаться между различными стратегиями во время выполнения, изменяя таким образом поведение алгоритма.


Назначение: Есть много похожих классов, отличающихся некоторым поведением? Стратегия вынесет отличающееся поведение в отдельную иерархию классов, а первоначальные можно будет свести  к одному.
Метафора: Вам нужно добраться до аэропорта. Можно доехать на автобусе, такси или велосипеде. Здесь вид транспорта является стратегией. Вы выбираете конкретную стратегию в зависимости от контекста — наличия денег или времени до отлёта
Потенциальные проблемы: Дополнительные классы = дополнительное место, где можно накосячить = усложнение программы. К тому же клиент должен знать, в чем разница между стратегиями.
